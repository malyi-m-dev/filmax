# Changelog

Все заметные изменения проекта. Формат близок к [Keep a Changelog](https://keepachangelog.com/ru/1.1.0/),
версии — по [SemVer](https://semver.org/lang/ru/).

Релизы выпускаются пушем git-тега `vX.Y.Z`: workflow `android-release.yml` собирает
подписанный APK и публикует его в [GitHub Release](https://github.com/malyi-m-dev/filmax/releases)
с авто-changelog из коммитов (Conventional Commits) между тегами. Этот файл —
кураторская выжимка поверх авто-генерации.

## [Unreleased]

## [1.6.1] — 2026-07-21

### Исправлено
- TV: холодный старт без сети показывал пустой чёрный экран — теперь объяснение
  с кнопкой «Повторить» (офлайн-баннер остаётся, когда есть что показать из кэша).
- Телеметрия слала событие на каждый сбой запроса: один запуск без сети давал пять
  non-fatal, и реальные баги тонули в них. В отчёты идут только сбои, похожие на баг
  (5xx, падения парсинга); нет сети, таймаут и протухшая сессия — хлебной крошкой.
  Debug-сборки не отправляют отчёты явным запретом, а не по стечению обстоятельств.
- Плеер: переключение на запасной вариант доставки потока больше не запускает
  воспроизведение, если оно было на паузе.

## [1.6.0] — 2026-07-21

### Добавлено
- Телеметрия ошибок (Firebase Crashlytics): non-fatal ошибки сети/парсинга с URL и кодом,
  ошибки воспроизведения, user id (username), версия приложения; debug-сборки не отправляют.
- Мобильный плеер догнал TV: кнопка «Следующая серия», автопереход в конце серии,
  спиннер буферизации, рабочая кнопка «Повторить» в модалке ошибки.
- Подпись версии приложения внизу Профиля (mobile и TV).

### Исправлено
- Плеер перебирает варианты доставки потока (hls4 → hls → http), когда CDN недоступен
  (DPI/SNI-блокировки api.srvkp.com) — лечит «следующая серия не запускалась».
- TV-плеер показывает карточку ошибки воспроизведения вместо немого чёрного экрана
  и спиннер во время буферизации.
- Выход из плеера кнопкой HOME больше не оставляет играющий звук: уход в фон ставит паузу.
- «Назад» с онбординга закрывает приложение; онбординг больше не встаёт поверх
  авторизованных экранов при протухании сессии.

## [1.5.0] — 2026-07-20

### Добавлено
- Постраничная догрузка контента: витрина Каталога (TV и телефон), ряды главной
  «В тренде», «Сериалы с высоким рейтингом» и «Подборки».
- Автостарт следующей серии на TV — через 5 секунд после появления плашки, не дожидаясь
  конца титров.
- Плавный кросс-фейд переходов между экранами.

### Исправлено
- Направление сортировки каталога было инвертировано: «Просмотры» и «Рейтинг» показывали
  хвост каталога вместо топа.
- Чип «Аниме» отдавал пустую выдачу: у kino.pub нет такого типа — это жанр поверх фильмов
  и сериалов.
- TV: возврат из карточки и переключение вкладок больше не теряют скролл, фокус и
  выбранный сегмент «Моё».
- TV: выбор сезона в карточке сериала сбрасывает ряд серий к началу.

## [1.4.1] — 2026-07-20

### Исправлено
- Экран гас во время просмотра при живом звуке.

## [1.4.0] — 2026-07-19

### Добавлено
- Автопереход к следующей серии на TV с обратным отсчётом и плашкой «Дальше».
- Выбранная озвучка запоминается на весь тайтл и применяется к следующим сериям.

## [1.3.0] — 2026-07-19

### Добавлено
- Обновления приложения из GitHub Releases (проверка версии, скачивание, установка).
- Голосовой ввод поиска внутри приложения.
- TV-плеер: панель сезонов и серий с прогрессом просмотра, видимый фокус транспорта.
- Полный список озвучек с названиями студий.
- Экран деталей на TV — единое полотно с двумя состояниями (hero → контент).
- Demo build type с зашитым токеном.

### Исправлено
- Запуск серии учитывает сезон: играет именно выбранный эпизод, а не первый с тем же номером.
- TV: вход фокуса в ряды всегда на первую карточку, мгновенное переключение разделов
  таб-бара, бегущая строка в подписях карточек.
- «Настройки устройства» скрыты из Профиля: `device/info` на бэкенде отвечает 500.
- Карточки «Моё» ведут в карточку тайтла, а не сразу в плеер.

## [1.0.0] — 2026-07-07

Первый публичный релиз. Приложение раздаётся как APK (Google Play не используется).

### Добавлено
- Релизный флоу: подпись release (`signingConfig` из `keystore.properties`/CI-секретов),
  R8 + `shrinkResources`, `versionCode` из числа коммитов и `versionName` из git-тега.
- CI по тегу `v*`: сборка подписанного APK → GitHub Release (+ Telegram).
